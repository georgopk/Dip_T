Hi,

very strange. Is it the right qmake? Not the one from Qt3? Try "which qmake" .

qmake's single purpose is to create the required makefiles and I have never seen a message like "nothing to be done"...

Concerning tinyxml: I have not used the git repo for it in a long time, tinyxml is availabe as a package in most Linux distributions nowadays... It would need to be installed to /usr for the latest openEMS version to find it.
Also, are there any plans to make an RHEL/CentOS package installation? We actually have a ROCKS installation http://www.rocksclusters.org/wordpress/ which is built on CentOS, and popular for computing clusters.
I don't have the knowledge nor the time to do it ...

I think compiling from source should be the best approach in most cases and I just have no idea why it is giving you so much trouble...
