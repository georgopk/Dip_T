your are not exciting a plane wave! Your explicitly excite a TE10 mode which is obviously not the same.

Post by thorsten » Tue 07 Feb 2017, 20:00

Is it done by the total-field, scattered-field (TFSF) technique (eg. this explanation)?
No not at all. This approach is possible too, look at the RCS_Sphere Tutorial.

The waveguide modes excite a mode according to the field-distribution provided. This can (usually) be a normal waveguide mode, or you can (misuse) it for a simple planar wave if you set the boundaries correctly.
1. Does openEMS automatically decide what kind of probes are approriate upon my "func_E" and "func_H"
field configurations?
Yes the mode functions are used to map the recorded fields onto them (mode matching). This produces a voltage and current as an integral (summation of matched electric/magnetic field times the mode function).
Furthermore you specify a cutoff wavenumber (kc, which is missing in your code snippet???).
From that calcPort can determine the waveguide impedance (for kc=0 --> 377 Ohms) which is used as for any port to calculate the incoming and reflected waves.
Of course you can specify any reference impedance to calcPort that you wish.

Does that answer the questions?
