Flaw in QCSXCAD.pro? Linker failed.

  • Quote

Post by darenw » Mon 05 May 2014, 07:38

When building QCSXCAD, the linker gave up at not finding libvtkCommon and cohorts. On my machine, the vtk .so files are in /usr/lib/vtk-5.10/ but apparently the OpenEMS build system couldn't figure that out. I made a quick hack fix to the .pro file, removing the logic for VTK_LIBRARYPATH and hard-coded the correct path. Yes, a wee bit short of professional software engineering best practicies, but it worked. I leave it to the OpenEMS developers to figure out the real problem and fix it properly.

Here's the diff for QCSXCAD.pro:

< isEmpty(VTK_LIBRARYPATH){
< } else {
< LIBS +=-L$$VTK_LIBRARYPATH
< }
---
> LIBS += -L/usr/lib/vtk-5.10
