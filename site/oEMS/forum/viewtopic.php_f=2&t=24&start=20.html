If you find an install instruction as I posted for you earlier, try not just to do what this list tells you. Try to understand what it does and why!
error: `physical_constants' undefined near line 18 column 1
This Octave error message should have clearly told you, that Octave couldn't find the function called "physical_constants" which means that Octave could not find the path to the matlab functions needed by openEMS.
So you could have deduced, that you either forgot the SetPath stuff or entered a false one.

Anyways, I hope that since openEMS now is up and running, you can get some good results.

Your next step really should be to understand how openEMS works... how the Matlab script interact with openEMS and how to setup the simulation and structures etc.

NEC is a method of moments code and therefor limited. It is very good at straight wires and is used by many radio amateurs for calculating their antenna patterns.
NEC is not useful if you want to incorporate dielectric materials or have compicated shapes. But it may be worth a look. Because of its limitations it is faster than openEMS (and any FDTD code in general).
openEMS is able to handle your problem. I've simulated coils myself and were very satisfied with the results and the agreement with measurements.

Sorry Thorsten, my fault.
Months ago when I coded octave_struct2xml_2.m, I set the float precision to an arbitrary value (9 in this case).
But the cylindrical multigrid code is very strict about the tolerance of the alpha values (homogeneity better then 1e-10).

The easy fix: open octave_struct2xml_2.m with an editor and change line 5:

Code: Select all

float_accuracy = 15;
This will be fixed in the next release.
