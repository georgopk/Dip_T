Hi,

this is indeed a difficult topic and for the future I should think about creating some static version.

One question? Are you able to install all dependencies? E.g. ask an admin to install the devel packages for e.g. hdf5, vtk etc?
If yes, just compile from source into your home-dir: http://www.openems.de/index.php/Compile_from_Source

If not, you may have to download all dependencies and build them first as well in your local directory... But that is a lot of work...

Using a VM would be possible, a tool like ldd would tell you all libs you would need to copy...
(compiling using my desktop computer running Ubuntu 12.04 didn't work as the libraries differ)
Maybe you copy all dependent libraries as well and use LD_LIBRARY_PATH to point openEMS to these instead?

Hi again,

I used my testing VM with Ubuntu 12.04 to grab openEMS and all dependencies using this script: http://h3manth.com/content/copying-shar ... pendencies

I only removed libc.so* and ld-linux-x86-64.so.2 and put it all into an archive:
http://www.openems.de/download/linux/op ... 64.tar.bz2

Maybe you give it a try. It worked on my Ubuntu 13.10, but that is newer than 12.04 and not older like your Debian.
But if it does not work you "simply" have to reproduce my steps using a Debian 6 VM...

I hope this helps, please give me some feedback on your findings...
