we use the latest hdf5 version (1.8.9). You need the define

Code: Select all

DEFINES += H5_USE_16_API
But it should be inside the pro-file. Maybe you compiled hdf5 without this "H5_USE_16_API" support?

But you are right. At some point in the future we should remove this necessary API. Why ever they introduced that and still use it in all their examples

Hi,

maybe to further simplify the process. You could also install git from source quite easy and put it e.g. in $Home/bin or something in your search-path.

Additionally you should be able to run qmake like this:

Code: Select all

qmake "LIBS += -L$HOME/install/lib" "INCLUDEPATH += <path/to/include>" openEMS_MPI.pro
Maybe this way you wouldn't need to modify anything?
