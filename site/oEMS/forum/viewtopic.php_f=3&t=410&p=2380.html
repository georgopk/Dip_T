Post by thorsten » Mon 12 Sep 2016, 19:07

Hi Anders,

transformations cannot be applied for lumped elements. What I have to fix is that no error/warning message is thrown...

The reason for this is that lumped elements are quite special, e.g. they have to be boxes oriented in Cartesian directions. Thus transformations make little sense. Of course simple scaling and translation would be possible, but that can be done easily with the start/stop coordinates already. That's why I never thought about transformations.

I hope it is ok for you to not be able to use transformations with lumped elements.

regards
Thorsten

Post by Hale_812 » Fri 16 Sep 2016, 05:01

AndersG, if possible, do not use transforms. Transforms are not properly handled by current VTK engine. Especially when trying to detect vertices for the mesh. Ind I suspect some degraded precision when applying rotation transform. I had a lot of pains with those, and finally gave up.

Instead, you have to define every object inside a function, where every coordinate matrix is modified with transformation matrices. The function definition thus becomes like [CSX, mesh] = AddMyObject (CSX, mesh, material, priority, position, rotation), where "mesh" .x/.y/.x object is populated directly by the function, with formulas defined by you.
