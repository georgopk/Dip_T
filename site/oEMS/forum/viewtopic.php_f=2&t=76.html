Post by thorsten » Fri 19 Oct 2012, 21:13

Hi,

I just tried a different option:

Add following line to openEMS.pro:

Code: Select all

QMAKE_LFLAGS += \'-Wl,-rpath,\$$ORIGIN\'
with this, openEMS searches for its libs in the same path (like Windows is doing it by default)

If you now copy all necessary or missing libs into your openEMS folder it should find them.

I was able to run openEMS like this on an Ubuntu Live-CD

Maybe this helps??

Post by sebastian » Fri 19 Oct 2012, 21:44

A cluster installation is special in that in normally mounts the home via nfs onto all cluster nodes. Then it makes sense to have a openEMS installation encapsulated inside a folder. For all other scenarios I can imagine, I would prefer the linux package manager and our binary packages.

I'm not sure if all linux variants accept the the RPATH and RUNPATH entries in the executables; LD_LIBRARY_PATH would be the rescue in such a case.

Give it a go.
