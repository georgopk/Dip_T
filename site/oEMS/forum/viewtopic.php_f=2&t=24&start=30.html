sebastian wrote:Sorry Thorsten, my fault.
Months ago when I coded octave_struct2xml_2.m, I set the float precision to an arbitrary value (9 in this case).
But the cylindrical multigrid code is very strict about the tolerance of the alpha values (homogeneity better then 1e-10).

The easy fix: open octave_struct2xml_2.m with an editor and change line 5:

Code: Select all

float_accuracy = 15;
This will be fixed in the next release.

Sebastian
Please do not change this float_accuracy to 15, it will create more problems:
For example, if I set a frequency f0=2.4e9 the xml file translation is around -1.89e9 which does not make sense.
It took me a few hours to find out. I change it back to float_accuracy = 9.

Post by sebastian » Fri 06 Apr 2012, 19:01

Version 3.2.3 is from Sep 2009. It is old and not supported. I know of at least one bug, which prevents using the openEMS optimizer with version 3.2.3.
Try to upgrade to at least 3.2.4, better to the current stable 3.6.1.
Some examples work only with a recent (>= 3.3.x) version of octave.
The header of the examples should state the tested octave version.

May be we should write a scentence about the required version at a prominent place.
