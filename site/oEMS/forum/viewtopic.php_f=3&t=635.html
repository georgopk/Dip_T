"WriteOpenEMS" is indeed creating the xml file. But the same file is just run by openEMS. Thus you can easily create it by a totally different means and run it with openEMS. Even completely outside of Octave (or Matlab).
Just run openEMS in a terminal with the xml file as the first argument and inside the folder in which to create the output.

The only thing you need to do then is the post-processing. But than can be done in Octave or any other software too as this is just reading and analysing the output files generated by openEMS...

May I ask by what means/tool you intend to create the xml files? And why?

Post by thorsten » Mon 02 Apr 2018, 09:35

Hi,

well still you could do all this always just (re-)creating the CSX-file with Matlab.
But there is an easier way maybe. openEMS supports discretized material files. They are used for e.g. human body models.
You could just create your basic CSX file and its mesh purely referencing such a discrete material file.
In all of your runs you only modifiy the material distribution in this material file and run openEMS with the same (unaltered) CSX file again...
The file is bases on the hdf5 file format and this can be modified in almost all languages...
I realize that this may not be a good topic for the forum
No, this is all about how to use openEMS. Simple or more complex. All is good and we should continue here ...
openEMS is all about doing crazy things that you potentially cannot do with other (e.g. commercial) tools.
