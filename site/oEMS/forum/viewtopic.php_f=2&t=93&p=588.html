Hi,

We use the openSUSE build service to create our binaries. Debian Wheezy is not yet present there.
Thorsten used a new feature from a recent tinyxml library, which is too old in Debian 6 and Ubuntu Natty. That's why the current openEMS is not yet available for these platforms.

To build your own package, go to
http://download.opensuse.org/repositori ... ntu_12.10/
and download the files ending in .dsc and .gz.
Then excecute

Code: Select all

dpkg-source -x *.dsc
Go into the csxcad directory and execute

Code: Select all

dpkg-buildpackage
This will produce .deb files that you can install with your prefered packaging tool (e.g. dpkg -i).

I hope that helps.

Sebastian

Well I just though about it.

I think running setup will fail? Because it needs to create an oct-file. But by default you cannot create files in /usr/share...
That was the reason why we needed a special package for that.
So either you have this h5readattr_octave.oct file already or you won't be able to create it...

Anyways running setup manually should not be necessary any more. If ReadHDF5Dump can't find the function it tries to execute setup automatically...

Possible solution, copy the openEMS interface into your home-folder (somewhere) and add this path to your scripts...
