Absorbing foam is not a PML. It is not matched, therefore pyramides are used for matching.
It is just a quasi-homogeneous lossy dielectric. The properties are provided by manufacturers and sometimes resellers provide with more detailed measurement results on your request.

But do not try to simulate this in OpenEMS - you will run out of RAM and die waiting.

And he is the only one qualified developer around for moving in that direction. There are papers on tetrahedral(like used in FEM) mesh solving, and multifold mesh density reduction for FDTD methods, but most people here (like me) are not qualified for simplest programming of solvers... so either we bring Thorsten some really appealing problem, or be happy with what we have.

Post by thorsten » Mon 30 Jan 2017, 20:44

In order to excite a plane wave from a rectangular port, which is the best function to use when using version 0.0.35? 'AddWaveGuidePort' or 'AddRectWaveGuidePort'?
First you need to specify what you want. Because you cannot excite a plane wave in an rectangular waveguide.

If you want a rectangular waveguide mode (e.g. TE01) you obviously need the AddRectWaveGuidePort, if you want to excite a plane wave you can use the AddWaveGuidePort which is the basis port for all of these types.
A AddRectWaveGuidePort is just a specialized version of an AddWaveGuidePort.
For the AddWaveGuidePort you have to specify the Mode of the wave using a formula (using x,y,z coordinates). Have a look at the "AddRectWaveGuidePort" implementation and you will see how it just calls AddWaveGuidePort using the already calculated mode for that special type of waveguide.

But now, what is the field distribution of a plane wave?
Well if you propagation is in z-direction you can e.g. have E_x and H_y for example (this would require PEC BC in x-direction and PMC in y-direction too).
That would give you:

Code: Select all

func_E = {1, 0, 0};
func_H = {0, 1, 0};
What would be kc (the waveguide cutoff number)? Well 0 of course as a plane wave does not have a cutoff.

Code: Select all

[CSX, port{1}] = AddWaveGuidePort(CSX, 0, 1, start, stop, 2, func_E, func_H, 0, 1);
That should do the trick and excite a plane wave at start and do the recording of the incoming and reflected waves at stop
Just make sure you do not put start or stop into the PML.

Agreed?
