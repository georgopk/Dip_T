Post by sebastian » Fri 16 Mar 2012, 19:46

Hello Georg Huber,

the command to add the scripting files is identical. Start octave and enter

Code: Select all

addpath('D:/openEMS/matlab');
(if openEMS was installed to D:/openEMS). The folder inside the openEMS install directory is named matlab, which is confusing if you use octave.

Octave will execute the script .octaverc inside your home directory (s.th. like c:\documents and settings\<username>). You may add the addpath to this file to have Octave execute it every startup.

We will consider adding a repository to share code, but in the meanwhile it is the simplest to create a new wiki page and post your code with explanations there. We are very glad to hear you like to share your code!

You're welcome.

Hi,

you may also want to have a look at Gui Octave (http://sites.google.com/site/guioctave/). It is a nice graphical front-end which makes using Octave much easier.

Keep in mind that Octave is kind of a free alternative to Matlab. It uses the Matlab Syntax etc. That's why the Matlab-Interface of openEMS (found in the folder openEMS/matlab) can be directly used with Octave as well.

Furthermore, after you added the Matlab/Octave Interface path (in your case D:/openEMS/matlab) as Sebastian described, you must execute:

Code: Select all

setup
in Octave to add some Octave specific functionality. But you only need to do this once for every new Octave or openEMS version you use.

If you want to use openEMS, your first step should be to make yourself familiar with the Matlab/Octave Syntax.
Maybe this Matlab Seminar (http://nts.uni-duisburg-essen.de/downlo ... r_main.pdf) is helpful. At least the first four chapters.

Post by thorsten » Wed 21 Mar 2012, 18:40

Hi,

I'm sorry to hear that this is so frustrating.

I can't quite make out where the problems have their root. So maybe I just write down what steps I just took on my Windows Notebook.

1) Download openEMS, Octave 3.2.4, Gui Octave
2) unzip openEMS to D:\openEMS
3) Install Octave, just accepting all defaults
4) Install GuiOctave, accepting all defaults
5) Running GuiOctave
6) File->Set Paths : Add folder... "D:\openEMS\matlab"
7) running "setup" in the command window: this will compile some Matlab functions needed by openEMS which Octave is not supporting by default
8) GuiOctave Editor: File->Open open a tutorial example
9) Press "Run" (green triangular icon) OR Debug->Run
10) Maybe close the AppCSXCAD windows after reviewing the structure
11) Enjoy the openEMS results

I hope this helps. It just worked like this for me. Maybe you can tell me at which point it failed for you.

regards
