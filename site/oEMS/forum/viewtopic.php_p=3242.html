Hi,

I inherited an openEMS Octave project, that I am converting to Python to allow for easier programmatic changes, and to get a working HDF5 implementation (as I understand it Octaves HDF5 is broken)

I am getting the following output when running (Python):
--- Excitation Extension ---
Voltage excitations : 0 (0, 0, 0)
Current excitations : 0 (0, 0, 0)
Excitation Length (TS) : 471
Excitation Length (s) : 9.55513e-11
--- Mur ABC Extension ---
Active direction : x at line: 0
--- Mur ABC Extension ---
Active direction : x at line: 454
--- Mur ABC Extension ---
Active direction : y at line: 0
--- Mur ABC Extension ---
Active direction : y at line: 454
--- Mur ABC Extension ---
Active direction : z at line: 0
--- Mur ABC Extension ---
Active direction : z at line: 207
-----------------------------------
And my probes all have zero signal.


The working Octave interface created an Excitation with:
%% setup FDTD parameter & excitation function %%%%%%%%%%%%%%%%%%%%%%%%%%%%%
FDTD = InitFDTD('NrTS', numTS, 'EndCriteria', 1e-05);
FDTD = SetGaussExcite(FDTD,0,f0);
BC = {'MUR','MUR','MUR','MUR','MUR','MUR'};
FDTD = SetBoundaryCond(FDTD,BC);
CSX = InitCSX();
CSX = DefineRectGrid(CSX, unit, mesh);
%% apply the excitation %%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
start = [0,0,(mesh.z(2)+30*mesh.z(3))/31];
stop = [0,0,(30*mesh.z(3)+mesh.z(4))/31];
CSX = AddExcitation(CSX,'excite',0,[1 1 0]);
weight{1} = '(x)/(x*x+y*y)';
weight{2} = 'y/pow(rho,2)';
weight{3} = 0;
CSX = SetExcitationWeight(CSX, 'excite', weight );
CSX = AddCylindricalShell(CSX,'excite',0 ,start,stop,0.5*(coax_rad_i+coax_rad_ai),(coax_rad_ai-coax_rad_i));

in Python:
csx = ContinuousStructure()
FDTD = openEMS(NrTS=numTS, EndCriteria=1e-5)
FDTD.SetCSX(csx)
FDTD.SetGaussExcite(0.0, f0)
BC = ('MUR','MUR','MUR','MUR','MUR','MUR')
FDTD.SetBoundaryCond(BC);

# part of an antenna class
def add_excitation(self):
"""
"""
start = self. coax_start + np.array([ 0.0 , 0.0 , (self.bowtie_center[2] - self.coax_start[2])/31.0 ])
stop = self. coax_start + np.array([ 0.0 , 0.0 , 2.0*(self.bowtie_center[2] - self.coax_start[2])/31.0 ])
excite = self.csx.AddExcitation('excite', exc_type=0, exc_val=[1, 1, 0] )
weight = [ '(x)/(x*x+y*y)' , 'y/pow(rho,2.0)' , '0.0'];
excite.SetWeightFunction(weight)
excite.AddCylindricalShell(start,stop,0.5*(self.coax_rad_i + self.coax_rad_ai),(self.coax_rad_ai - self.coax_rad_i))

The missing excitation can have multiple reasone. Maybe the mesh is not the same or the weighting does not work...
I think dumping the CSX to xml and compare both would be good. Yes with python the openEMS settings are missing, but they are simple and easy to compare otherwise.
The best approach is in Octave to use "--debug-CSX". This way the files are created the same way and be easier to compare...
Please attach both files.
