Updates on the brew installation of openEMS:

- after getting it to install, you'll need to manually make an openEMS.sh wrapper for the openEMS binary (the brew install doesn't install it), and symlink it to the appropriate place (probably /usr/local/bin for standard home brew installs)


Create /usr/local/Cellar/openems/HEAD/bin/openEMS.sh with the following:

#!/bin/bash
BIN_PATH=`dirname $0`
# unset the library search path
# Matlab sets it to point to an old libc
export LD_LIBRARY_PATH=
$BIN_PATH/openEMS $@

Save it, then symlink it.

# cd /usr/local/bin
# ln -s ../Cellar/openems/HEAD/bin/openEMS.sh openEMS.sh

Now searchBinary() can find the file.


In addition, >> CSXGeomPlot('nonexistant.xml') fails:

invoking AppCSXCAD, exit to continue script...
/usr/local/Cellar/openems/HEAD/share/CSXCAD/matlab/../../../bin/AppCSXCAD.sh: line 15: /usr/local/Cellar/openems/HEAD/share/CSXCAD/matlab/../../../bin/AppCSXCAD: No such file or directory

This is because AppCSXCAD is built as a Mac OS X app bundle. Edit /usr/local/Cellar/openems/HEAD/bin/AppCSXCAD.sh and change the last line from:
$BIN_PATH/AppCSXCAD $@
to:
$BIN_PATH/AppCSXCAD.app/Contents/MacOS/AppCSXCAD $@

Now >> CSXGeomPlot('nonexistant.xml') fails thusly:
